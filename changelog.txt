
RuntimeX 5.1.1

Improved
* LibRtX is now the only injection framework.
* Better ARM-on-x86/x86_64 emulator loading through Android Runtime native load
  before lower-level native-bridge fallbacks.
* Better Android virtual-environment handling for VMOS, VPhoneGaGa, and
  Virtual Master by keeping ABI selection on native ARM paths.
* Target-app staging is required, reducing misleading dlopen path errors.
* Simpler logs with clean HH:mm:ss format and errors marked with "!".

New
* In-app updater with changelog, APK download, and install flow.
* Online ModMenu delivery using modmenu.json direct links for all four ABIs.
* Startup and Settings now check RuntimeX and ModMenu updates together.
* Activate now offers Runtime-only or Runtime + ModMenu download.

Cleanup
* Removed old secondary-backend build paths, memfd settings, and raw ZIP bundling.
* Removed embedded ModMenu binaries from the APK.


RuntimeX 5.0.1 - Beta

New
• RtxInject injection framework — a second, ptrace/KittyMemory-based injector
  alongside the Frida-based LibRtX. Pick it in Settings → Injection framework.
  Broader emulator coverage, with automatic memfd fallback and retries for
  tricky boxes like vPhoneGaGa.
• In-app updater — check for updates, view the changelog, download to your
  Downloads folder, and install with one tap.

Improved
• Simpler logs — clean HH:mm:ss format, errors marked with "!".
• Logs auto-clear on every launch and for each game you inject, so you only
  ever see the current session.

Fixes & housekeeping
• Injection stability improvements across emulators.
• Cleaned up the project and added a proper changelog.
